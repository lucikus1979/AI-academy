# Day 1 Glossary

## Core Concepts

| Term | Definition |
|------|------------|
| **LLM** | Large Language Model - AI trained on text to predict next words. Examples: GPT, Claude, Gemini, Llama. |
| **GPT** | Generative Pre-trained Transformer - OpenAI's LLM architecture. Current version: GPT-5.2. |
| **Token** | A piece of text (roughly 4 characters or 0.75 words). LLMs process text as tokens. |
| **Prompt** | The input text you give to an LLM. What you ask or tell it. |
| **Completion** | The output text generated by an LLM in response to a prompt. |
| **Context Window** | Maximum text an LLM can process at once. Measured in tokens (e.g., 128K tokens). |

---

## AI Agents

| Term | Definition |
|------|------------|
| **Agent** | An LLM that can take actions (call APIs, run code, browse web), not just generate text. |
| **Autonomous Agent** | An agent that can operate with minimal human oversight, making decisions independently. |
| **Multi-Agent System** | Multiple agents working together, each with specialized roles. |
| **Orchestration** | Coordinating multiple agents to work together on complex tasks. |
| **Tool Use** | Ability of an agent to call external functions (APIs, databases, services). |
| **Handoff** | When one agent passes work to another agent in a multi-agent system. |

---

## Prompting & Behavior

| Term | Definition |
|------|------------|
| **System Prompt** | Hidden instructions that shape LLM behavior. Defines role, rules, and constraints. |
| **Hallucination** | When AI generates false but confident-sounding information. |
| **Grounding** | Connecting LLM to factual data sources to reduce hallucinations. |
| **Chain-of-Thought** | Prompting technique to improve reasoning by asking model to show its work. |
| **Temperature** | Parameter controlling randomness of outputs. Low = deterministic, High = creative. |
| **Constitutional AI** | Anthropic's approach to making AI follow principles and have "character". |

---

## RAG & Data

| Term | Definition |
|------|------------|
| **RAG** | Retrieval-Augmented Generation - connecting LLM to external data to answer questions. |
| **Embedding** | Numerical representation of text for similarity search. Turns words into vectors. |
| **Vector Database** | Database optimized for storing and searching embeddings. Examples: Pinecone, Weaviate. |
| **Semantic Search** | Finding information by meaning, not just keywords. |
| **Chunking** | Breaking documents into smaller pieces for RAG processing. |

---

## Training & Customization

| Term | Definition |
|------|------------|
| **Fine-tuning** | Adapting a pre-trained model on specific data to specialize it. |
| **Pre-training** | Initial training of LLM on massive text datasets. |
| **RLHF** | Reinforcement Learning from Human Feedback - technique to align models with human preferences. |
| **Inference** | Using a trained model to generate outputs (as opposed to training it). |

---

## Agent Patterns

| Term | Definition |
|------|------------|
| **ReAct** | Reasoning + Acting pattern. Agent thinks, then acts, then observes, then thinks again. |
| **Planning Agent** | Agent that creates a plan before executing, breaking complex tasks into steps. |
| **Reflection** | Agent evaluating its own output and correcting mistakes. |
| **Memory** | Agent's ability to remember context across conversations or sessions. |

---

## Enterprise & Governance

| Term | Definition |
|------|------------|
| **KAF** | Kyndryl Agentic AI Framework - Kyndryl's enterprise platform for AI agents. |
| **Guardrails** | Safety mechanisms that prevent AI from harmful or unintended actions. |
| **Red Teaming** | Testing AI systems by trying to make them fail or behave badly. |
| **Audit Trail** | Record of all agent actions for compliance and debugging. |
| **EU AI Act** | European Union regulation on artificial intelligence (effective 2025-2026). |
