6 Prompt Engineering Basics

How to communicate effectively with AI systems.

🎯 Be Specific

Vague prompts get vague answers. State exactly what you need.

📋 Provide Context

AI doesn't know your situation. Give background and constraints.

🎭 Define the Role

Who should the AI be? A consultant? A code reviewer? A teacher?

📐 Specify Format

List? Table? JSON? Code? Tell the AI what output shape you want.

Bad vs Good Prompt

❌ Bad Prompt
Tell me about AI agents.

Too vague. What aspect? For whom? What format?

✅ Good Prompt (using Markdown)
# Role You are an AI consultant explaining technology to a bank CEO. ## Task Explain AI agents in **3 paragraphs**: 1. What they are *(no jargon)* 2. One concrete banking example 3. Key risk to be aware of ## Constraints - Keep it under `200 words` - The CEO has **2 minutes** to read - Use simple language ## Output Format ``` **Summary:** [one sentence] **Example:** [banking use case] **Risk:** [main concern] ```

Markdown headers for structure • Bold/italic for emphasis • Lists for clarity • Code blocks for output format

System Prompt Structure

Every agent starts with a system prompt. Here's the template:

system_prompt.md
# Agent Name

## Role
You are a [specific role] that [primary function].

## Capabilities
- What the agent CAN do
- Another capability

## Constraints
- What the agent CANNOT do
- Limitations to enforce

## Output Format
[Structured template for responses]
💡

Pro tip: The best system prompts define what the agent should NOT do. Boundaries matter more than capabilities.

7 Markdown Basics

The universal language for AI documentation. AI models understand Markdown natively.

✅ Plain text

Works everywhere — no special software needed.

✅ Easy to learn

5 minutes to learn the basics, productive immediately.

✅ Version control

Git-friendly — diffs show meaningful changes.

✅ AI-native

All major models understand and produce Markdown.
